# 云原生与部署运维

## 栏目定位

本栏目关心代码离开 IDE 之后的一切——它如何被打包、被部署、被编排、被监控，以及跑起来之后如何保持稳定高效。

**一句话描述：** 代码从提交到上线的整个生命周期，以及支撑系统的底层基础设施。

## 内容范围

| 子领域             | 示例主题                                                     |
| ------------------ | ------------------------------------------------------------ |
| 基础设施与操作系统 | Linux 内核调优、Shell 脚本技巧、系统故障排查（磁盘满、CPU 高、OOM） |
| 中间件运维与原理   | Kafka/RabbitMQ 集群部署与监控、MySQL/Redis 主从与备份、Nginx 反向代理配置 |
| 容器与编排         | Dockerfile 最佳实践、多阶段构建、Kubernetes 核心对象与管理、Helm Chart 编写 |
| CI/CD 与交付流水线 | GitHub Actions/GitLab CI 实战、多环境管理、灰度发布与蓝绿部署策略 |
| 可观测性与排障     | 日志系统（ELK/Loki)、指标监控（Prometheus+Grafana)、链路追踪、线上排障复盘 |

## 写作原则

- **动手即真理：** 所有配置和流程必须经过实际验证。
- **全链路视角：** 排障类文章要展示从现象到根因的完整推导过程。
- **版本标注：** 所有中间件和工具必须标注版本号，避免过时误导。

## 与其他栏目的边界

- 若文章重点是“Kafka 消费者在 Go 中如何实现幂等消费”，归 **工程与架构实践**。
- 若文章重点是“Go 的 PGO 编译优化”，归 **Golang 修炼之道**。
- 若文章重点是“我推荐哪些终端工具”，归 **工具与资源**。
