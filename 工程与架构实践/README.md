# 工程与架构实践

## 栏目定位

本栏目是博客的核心竞争力所在——关注代码怎么写、项目怎么组织、系统怎么设计，覆盖从单仓库规范到分布式架构的完整工程实践图谱。

**一句话描述：** 让代码从“能跑”到“可维护、可扩展、可交付”的一切实践，都在这里。

## 内容范围

| 子领域             | 示例主题                                                     |
| ------------------ | ------------------------------------------------------------ |
| 项目规范与代码质量 | 目录结构设计、错误处理规范、单元测试与 Mock、代码评审清单    |
| 设计模式与架构     | 微服务拆分、领域驱动设计（DDD)落地、事件驱动架构、CQRS       |
| 最佳实践           | 幂等性设计、分布式事务方案、缓存与数据库一致性、限流熔断降级 |
| 高频踩坑与复盘     | 线上事故复盘、并发竞态问题、内存泄漏排查、数据不一致修复     |
| 数据库与存储设计   | 表结构设计范式与反范式、索引优化案例、分库分表落地、SQL 调优实战 |

## 写作原则

- **场景带入：** 从真实业务场景出发，而非凭空讲理论。
- **方案对比：** 对于同一个问题，给出多种解法并分析优劣。
- **可复现：** 核心案例应提供最小复现示例。

## 与其他栏目的边界

- 若文章重点是“MySQL 索引的数据结构原理”，归 **云原生与部署运维**。
- 若文章重点是“GORM 这个 API 怎么用”，归 **Golang 修炼之道**。
- 若文章重点是“如何与产品经理沟通需求”，归 **技术人文与软技能**。
