# AI-Coding 时代

## 栏目定位

本栏目记录我作为后端工程师，如何拥抱 AI 辅助编程浪潮——不只是“用 AI 写代码”，更是“如何用 AI 写更好的代码”。

**一句话描述：** 探讨 AI 如何重塑编程范式，以及一个现代程序员的 AI 工具链实践。

## 内容范围

| 子领域            | 示例主题                                                     |
| ----------------- | ------------------------------------------------------------ |
| AI 辅助编程实战   | Copilot/Gemini Code Assist 使用技巧、用 AI 生成单元测试、AI 重构遗留代码 |
| Prompt 工程       | 面向代码生成的 Prompt 设计模式、上下文工程、System Prompt 调优 |
| AI 工具评测与选型 | Cursor vs Copilot、Codeium 体验、不同 LLM 在编程任务上的表现对比 |
| Agent 与自动化    | 用 AI Agent 完成端到端开发任务、AI 驱动的代码审查、自动化文档生成 |
| 范式思考          | AI 时代程序员的核心竞争力、AI Coding 对软件工程的影响        |

## 写作原则

- **亲测有效：** 所有技巧都来自本人的真实使用体验，并附上前后对比。
- **可复现：** Prompt 和配置要完整公开，读者可以照做。
- **批判性思维：** 不盲目吹捧，记录 AI 的局限和踩坑场景。

## 与其他栏目的边界

- 若文章重点是“用 Go 实现一个算法题”，即使用了 Copilot 辅助，仍归 **Golang 修炼之道**。
- 若文章重点是“用 AI 优化 CI/CD 流程”，归 **云原生与部署运维**。
- 本栏目侧重“AI 如何改变编程方式”这一视角本身。
