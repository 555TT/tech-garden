# 技术人文与软技能

## 栏目定位

技术之外，还有一个程序员的成长轨迹、协作智慧和职业生涯思考。本栏目是博客的“人文底色”，记录我在技术道路上的非代码感悟。

**一句话描述：** 成为一个更好相处的、更有影响力的、职业生涯更长的技术人。

## 内容范围

| 子领域           | 示例主题                                                     |
| ---------------- | ------------------------------------------------------------ |
| 技术管理与领导力 | 技术方案评审艺术、如何带新人、技术负债管理、跨团队协作       |
| 沟通与协作       | 如何与产品经理 battle、技术方案文档写作、代码评审中的沟通技巧 |
| 职业规划与成长   | 技术人成长路径选择、大厂 vs 创业、技术深度与广度的平衡       |
| 技术写作与表达   | 如何写好技术博客、技术演讲准备心得、画好一张架构图           |
| 效率与心流       | 深度工作方法、信息过载应对、技术人的时间管理、情绪与心力管理 |

## 写作原则

- **真诚而不说教：** 从个人经历出发，分享得与失，而非“我教你做事”。
- **引发的启发大于结论：** 鼓励读者思考自己的处境，而非照搬方案。
- **长期价值：** 文章的生命周期应超过具体技术版本，追求常读常新。

## 与其他栏目的边界

- 若文章重点是“如何用 Markdown 画时序图”，归 **工具与资源**。
- 若文章重点是“某个具体架构方案的选型决策过程”，归 **工程与架构实践**。
- 本栏目侧重“人”的成长与协作，而非“技术”本身。
