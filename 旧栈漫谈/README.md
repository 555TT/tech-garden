# 旧栈漫谈

## 栏目定位

本栏目收录我在 Java 时代的经验总结与技术思考，是我技术成长路径的见证。

这些内容不再代表我当前的技术重心，但其中沉淀的工程思维和架构理念，
至今仍在影响我写下的每一行 Go 代码。

**一句话描述：** 过去写下的字句，铺成了今天走向 Go 的路。

## 内容范围

- Java 核心知识总结（JVM、多线程、集合框架）
- Spring 生态实践经验
- Java 项目中的架构与设计复盘
- 从 Java 到 Go 的迁移心得与语言对比

## 写作原则

- **不追求高频更新：** 本栏目为存量整理，非活跃写作方向。
- **价值过滤：** 只收录今天重读仍有启发的文章，过时内容不再搬运。
- **读者预期管理：** 每篇文章均在文首标注原始写作时间和背景。

## 与其他栏目的边界

- 若文章虽是 Java 实现，但核心讲通用架构设计，优先归 **工程与架构实践**。
- 本栏目侧重“特定于 Java 生态、难以迁移他处”的经验总结。
