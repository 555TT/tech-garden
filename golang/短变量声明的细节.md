# :=短变量声明的细节

:=是声明变量和赋值变量的简写形式。num:=1等价于

```golang
var num int
num = 1
```

但是:=使用有几个条件：

* 必须在函数内使用，不能用于声明包级变量

```go
num := 1 //这样是错误的
func main{
    
}
```

* :=左边可以有声明过的变量，但必须有一个新变量。当:=左边的变量有声明过变量的时候有两种情况

①在同一作用域

```go
	a, b := 1, 2
	fmt.Printf("%p,%p\n", &a, &b)
	c, b := 100, 200
	fmt.Printf("%p,%p\n", &c, &b)
	//两个b的内存地址相同。结论；同一作用域，重新赋值
```

②在不同作用域。for、if等

两个例子：

```go
	a, b := 1, 2
	fmt.Printf("%p,%p\n", &a, &b)
	if true {
		c, b := 100, 200
		fmt.Printf("%p,%p\n", &c, &b)
	}
	//两个b的内存地址不同，不同作用域，是重新声明
```

```go
	a, b := 1, 2
	fmt.Printf("%p,%p\n", &a, &b)
	{
		c, b := 100, 200
		fmt.Printf("%p,%p\n", &c, &b)
	}
```

b在if和裸{}作用域中并不存在，是新声明，它遮蔽了外层的b。并且作用域结束后，b变量会消失。

一个奇怪现象：

for循环中使用:=

```go
for i:=1;i<10;i++{
    fmt.printf("%p\n",&i)
}
```

结果是每次打印的i是一个新的内存地址。很多人就误以为是每次都声明了一个新的i变量。其实不然，

## 总结

短变量声明规则

在同一作用域中:= 左边 至少有一个是新变量；已存在的变量会被**重新赋值（不是重新声明）**

作用域不同 ⇒ 就是不同的变量
