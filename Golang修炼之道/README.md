# Golang 修炼之道

## 栏目定位

本栏目聚焦 Go 语言本身，从语法特性到运行时原理，从并发模型到性能优化，记录我在 Go 语言上的深度探索。

**一句话描述：** 凡是涉及 `import` 语句右侧的库、以及 `go build` 背后的原理，都归这里。

## 内容范围

| 子领域             | 示例主题                                                     |
| ------------------ | ------------------------------------------------------------ |
| 语言基础与进阶     | 接口与反射、泛型实战、类型系统、内存对齐                     |
| 并发编程           | Goroutine 调度、Channel 高级模式、sync 包源码解读、并发 Bug 排查 |
| 标准库与主流三方库 | net/http 源码剖析、GORM 高级用法、sqlx 连接池调优            |
| 编译与运行时       | 逃逸分析、GC 机制、pprof 性能分析、编译优化与交叉编译        |
| Go 版本演进        | Go 新版本特性解读、版本升级踩坑记录                          |

## 写作原则

- **代码驱动：** 核心论点必须有可运行的代码示例支撑。
- **深度优先：** 拒绝浮于表面的 API 罗列，追求“为什么这样设计”的本源。
- **实战导向：** 结论必须来自真实项目中的踩坑或性能优化经历。

## 与其他栏目的边界

- 若文章重点是“如何用 Go 调用 Kafka”，归 **云原生与部署运维**。
- 若文章重点是“如何组织 Go 项目的目录结构”，归 **工程与架构实践**。
- 若文章重点是“我用 Copilot 生成了一段 Go 代码”，归 **AI Coding 时代**。
