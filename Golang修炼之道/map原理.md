# go map原理

golang从1.24开始引入swiss算法等来优化map实现，所以在1.24之前的版本中map的源码定义基本都在runtime/map.go中，从1.24开始，你就要看runtime/map_noswiss.go文件了。



makemap():

这个方法的t参数是干嘛的

在 Go 源码中，map 相关的函数经常会有一个 t 参数，这个 t 一般是 *rtype 或 *maptype 类型，代表 map 的类型信息。它用于描述 map 的 key、value 类型、大小、哈希函数等元数据。
在 runtime 层，t 参数通常用于：
获取 key/value 的类型和大小
选择合适的哈希算法
进行类型断言和反射操作
分配和管理 map 的内存布局
简单来说，t 就是“这个 map 的类型描述”，让底层代码能处理任意类型的 map。

关于研究源码的建议：一定要仔细看每个方法、每个参数、任何地方的注释，虽然是英文的，可能读不懂，看不懂也要看，这些都是官方源码作者的注释，它既然被写到那里就一定有用，真看不懂复制到翻译软件去翻译也要看。
